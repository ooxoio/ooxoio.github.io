数据结构：记录常见的数据结构。

堆（优先级队列）

1.性质：

  • 是一棵完全二叉树，最后一层不一定满。
  • 任意一棵子树的根节点大于/小于它的所有子节点。

2.分析：

优先级队列是以堆为底层再封装的，两者性质相同。因此这里以堆来说明。

通常实现堆（包括STL），通常会使用数组，数组首位就是堆顶元素。

因为堆是棵完全二叉树，那么父节点和左右孩子的结点下标就有了规律：

假设数组下标从0开始使用，从1还是从0开始使用只是编码的时候细节方面不同而已。

父节点下标为x，则左孩子下标 = 2 x + 1，右孩子下标 = 2 x + 2。

孩子下标为x，则父节点下标为2 / x。

因此，使用数组可以方便的模拟堆第一条性质这个结构特点。

对于第二条性质，就需要我们在对堆做增、删、改的时候提供相应的调整策略。

3.实现：以大顶堆为例

3.1建堆：
有两种方案，自上而下，自下而上。

自上而下：无论堆结点数是否已知，都适用。

每插入一个结点（在数组里就是末尾增加一个元素），就和它的父节点比较，如果大于父节点，向上交换，并且继续向上比较。

时间复杂度：O(NlogN)。

自下而上：只适于堆结点个数已知的情况。

这里按照数组下标从0开始，一共N个结点。

结合堆的第一条性质可以知道，最后一个非叶子结点的下标：N / 2 - 1。

从该结点开始，往前遍历，每次将该结点和左右孩子比较，左右孩子里最大值如果大于当前结点，向下交换，并且继续向下比较。

时间复杂度：O(N)。

分析两种策略的时间复杂度：

假设N个结点，树高 h = log N，k = 当前结点的树高。

自上而下：

每次从根节点开始建堆，每个结点最多比较次数是 h - k。也就是说根节点比较次数为0，因为刚开始就它这个结点，而下一层即k = h - 1的结点最多比较1次。

也就是说：这种策略下，树高越高，越靠上的结点比较次数越少。而二叉树 树高越底、越靠下的结点个数越多！

对于树高等1的叶子结点来说，它的比较次数最高可以达到h = log(N)次 ，并且该层的结点数量（这里假设是满二叉）是2^(h - 1) 。也就是说最底层的比较次数就高达 log N * 2^(h - 1)。

根据时间复杂度忽略低阶 ，最底层的个数2^(h-1)是最高阶的，对应的操作次数logN也是最大的，所以其实可以直接忽略树高比较高的那些结点的操作次数，就是等于 log N 2^(h) = logNN。

如果非要计算的话，也可以按下面的2S - S的方法。

其实很多复杂度的计算都类似这样，根节点操作次数少，树高越低的操作次数越多，而此时树高越低的结点数也更多，高阶的完全可以忽略前面那些低阶的操作次数，因此时间复杂度为N*logN。

3.2堆排序：建完堆后即可利用第二条性质来进行排序。

上面我们假设是大顶堆，那么这里就是升序。

每次将堆顶元素（即最大值）与末尾元素交换，那么最大值位于数组末尾，但交换后可能破坏了堆的结构，需要对堆顶元素向下比较交换重新使其满足堆的性质。

经过N - 1轮后即排序结束。

时间复杂度：O(NlogN)。和自上而下建堆一样的，越靠后的比较次数越高，所以累加起来会达到NlogN。
