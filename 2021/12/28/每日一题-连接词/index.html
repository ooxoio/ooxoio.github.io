LC.153.连接词

给你一个 不含重复 单词的字符串数组 words ，请你找出并返回 words 中的所有 连接词 。

连接词 定义为：一个完全由给定数组中的至少两个较短单词组成的字符串。

示例 2：

提示：

  • 1 <= words.length <= 104
  • 0 <= words[i].length <= 1000
  • words[i] 仅由小写字母组成
  • 0 <= sum(words[i].length) <= 105

思路

第一眼看这个题目的时候，感觉有点无从下手，尤其是它数据量也很大，最多O(N²)。

根据连接词的概念提取些关键词出来，组成，连接，前缀。当想到前缀树字典树

的时候，就稍微有点思路了。

我们可以将这些词先构造成一棵字典树，然后再让每个词在字典树上跳转，判断是否是连接词。具体跳转方式就是，如果当前走到的这个结点是某个单词的结尾，说明这个短词可能是我的组成一部分，我就跳转到头部继续看下一个词会跳到哪（即递归深搜)。最终如果跳完了，并且过程中经历了至少两次跳转到头部，说明这个词是连接词。当然，如果中途匹配失败，直接返回false,跳到上一层递归继续判断。

编码

优化：

1.先将words按长度升序排序后插入字典树。

2.边插入边判断是否是连接词，如果是则无需将该词插入，因为它可以用更短的连接词替代。（这一步很重要，不然极端情况下会超时，比如”a”,”aa”,”aaa”….)

3.可以不记录cnt，因为我们采取上述策略的话，我们在判断words[i]是否是连接词的时候，它还没插入字典树，并且题目保证words不存在重复单词。也就是说只要words[i]能走到末尾，就说明它肯定是经历了两次以上的更短的词组成而来的。

其它的解题思路：

同样按长度升序排序。

1.字符串哈希，将每个字符串哈希成一个数字。

2.words[i] = s, dp[i] = k含义是：字符串的前i个字符组成的字符串，最多能由words中k个其它单词组成。

3.转移方程：dp[i] = max(dp[j] (j < i && j = [0,i - 1])) + 1 。（前提，dp[j]存在，并且 [j + 1, i - 1]这个单词也存在）
